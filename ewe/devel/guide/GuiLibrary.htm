The GUI Library – Simple Controls

 

Text Display Controls

These are controls that are used to display text but do not provide any editing capabilities.

mLabel

This is a single line label.

 

ü      You can set the control field to refer to another Control on the display. If you do this, pen/mouse presses on the mLabel will be passed on to the Control. For example, if you have a label for a text editing control, clicking on the label will shift focus to the editing control.

MessageArea

This is a multi-line label (it inherits from mLabel). Use setText() to set the text for the control with each line of text being separated by \n (newline) characters.

TextMessage

This is a multi-line text display that automatically wraps its text to fit in the space allocated to the control. It fully implements ScrollClient, so it can be placed in a ScrollBarPanel.  This is the best control to use when you have a lot of information to display (e.g. a help file).

Text Edit Controls

These are controls that inherit from the abstract EditControl class.

ü      You can set the validator field to an InputValidator to validate keys as they are being entered.

mInput

This is a single line input control.

ü      You can set the isPassword field true to display only * characters during input.

mTextPad

This is a multi-line input control. It will automatically scroll within its area but it will not display scroll bars. You have to place it in a ScrollablePanel (e.g. a ScrollBarPanel) to utilize the scroll bars. This control also allows wrapping of lines to fit within the display and allows the use of shifted cursor keys for clipboard operations.

DateDisplayInput

This control is used to display and enter a date/time value. Entering of a Date is done via a pop-up calendar like interface, unless there is not enough display area, in which case it is entered via a single input line.

 

Buttons and Checkboxes

mButton

This is the basic button object. This type of button can display an image along with text. If you are using an image with text, you should set the textPosition field to be mGraphics.Up if the text is above the image, or mGraphics.Down if the text is below the image.

mCheckBox

This is the basic checkbox/radio button object. You can retrieve the state of the checkbox by using getState() and you can set the state using setState(boolean).

To make the checkbox one of a set of mutually exclusive radio buttons create a CheckBoxGroup object and call setGroup(CheckBoxGroup) on each of the checkboxes to go into the group.

Note that when you are using CheckBoxGroups, pressed events which are generated by each mCheckBox are captured by the CheckBoxGroup and the target of the event is changed from referring to the originating mCheckBox, to referring to the CheckBoxGroup. That is, the events now seem to originate from the CheckBoxGroup instead of originating from an individual mCheckBox.

ButtonCheckBox

This is a type of mCheckBox which is displayed as a normal button. When its state is true it is displayed as being in the depressed state. You can use ButtonCheckBox objects with CheckBoxGroup objects as well.

Choices, Menus and Lists

These all inherit from the abstract ChoiceControl class which allow a user to select one of a list of possible options. The mList control also has the ability to allow the user to select more than one item out of the list.

 

A ChoiceControl has a items member which is a Vector containing each selectable item in the control. All the methods that add items to the control will add to this Vector. You can add any one of two types of objects to this vector. You can add String objects, or you can add MenuItem objects.

 

A MenuItem is mostly used for Menu controls only. They allow for sub-menus, separators and also for individual items to be disabled or checked. The MenuItem label field holds the text which is to be displayed in the list.

 

The following methods should be used to manipulate the items in a ChoiceControl. None of them (except for changeItem()) will call a repaint of the control.

 

removeAll() – This clears the list of options.

addItem(Object item) – This adds an individual item. The item itself may be a String or a MenuItem object.

addItems(String []) – This adds a set of string items.

insertItemAt(Object item,int index) – This adds an item at a particular index.

changeItem(int index,String newText) – This changes the text associated with the item at the index.

deleteItem(int index) – This deletes the specified item.

 

The following methods should be used to query the status and items in a ChoiceControl.

 

getItemAt(int index) – This returns a MenuItem which represents the item at the specified index. If you added the items for the ChoiceControl as MenuItem objects, then this will return the MenuItem you added at that index. If you added the items for the ChoiceControl as Strings then a dummy MenuItem object is returned with its label field set to the String which you added at that index.

getSelectedItem() – This returns the Object added at the selected index (either a String or a MenuItem). It returns null if no item is selected.

indexOf(Object item) – This returns the index of the item in the control. The item can be a String or a MenuItem.

mChoice

This is a simple dropdown choice control. It displays a line of text and a drop-down button on the right. Pressing the pen on the line of text will cycle the text to the next item in the list. Pressing the pen on the drop-down button will display a menu of selectable items.

ü      If you set dropDownButton to false the drop-down button will not be displayed.

ü      If you set alwaysDrop to true the drop-down list will be displayed even if you press on the text area.

Use setText() and getText() to get the text representation of the current selection. You can also use select(int index) to select a particular item index, and you can access the selectedIndex field to retrieve the index of the currently selected item. If no item is selected, selectedIndex will be –1.

Menu

This is a multi-level menu control. The individual items on the menu may have sub-menus, may be disabled and may be checked. See the separate chapter on Using Menus and Lists for more details on these advanced features. Whenever you add String items to a Menu they are automatically converted to new MenuItems.

 

Menus will fire a MenuEvent with the type set to MenuEvent.SELECTED when an item is selected. The selectedItem field in the MenuEvent specifies which item was selected (this may be a String or a MenuItem – depending on the control which generated it and what was added).

mList

This is a standard list box which inherits from Menu. It can be set to allow multiple selections by setting multiSelect to true. mList controls generate ListEvent objects (which inherit from MenuEvent) but also generate events when items are unselected, in which case the type is set to ListEvent.DESELECTED.

MenuChoice

This is an mChoice control which allows a full menu (with submenus if necessary) its drop-down  menu. This is in contrast to an mChoice which only allows a flat list for its drop-down menu.

mComboBox

This is an mInput field combined with an mChoice field to provide a control which allows the user to enter arbitrary text, or to select text from a list. You have access to both the mInput (input field) and mChoice (choice field) of the ComboBox.

MenuComboBox

This is a ComboBox with the choice field set to a MenuChoice control instead of a plain mChoice control.

PullDownMenu

This is a button (drawn flat normally) which when pressed, displays an associated menu. PullDownMenus will work at the bottom or top of the screen. They will automatically fit the menu onto the screen.

SimpleList

This is a list that does not use a Vector object to store the items in the list. It allows you to create lists of arbitrary length without worrying about running out of memory. The idea is that you override the methods which retrieve the individual list items which are to be displayed. To implement a SimpleList you should override the following methods:

ü      int getItemCount() – This should return the number of items in the list.

ü      String getDisplayItem(int index) – This should return a String representing the item to display in the list at the specified index.

ü      Object getObjectAt(int index) – This should return the Object data representing the item at the specified index.

Dynamic Controls and MultiPanels

SingleContainer

This is a Container that can hold a single control (which itself may be a container) and can change the control it contains dynamically using a setControl() method. This is provided because changing the contents of a Panel or CellPanel is not a simple procedure.

MultiPanel

This is actually an interface that allows multiple controls to be placed in the same area in a container, where only one of the controls is visible at a time. The control set which is visible at any time is selected programmatically through the MultiPanel interface methods, or via user input (e.g. by selecting a Tab in an mTabbedPanel).

CardPanel

A CardPanel is a full implementation of a MultiPanel which has no user methods for selecting which item to display. This can only be done programmatically.

mTabbedPanel

This is a MultiPanel which allows users to select which item to display by selecting on of a number of tabs.

The GUI Library – Complex Controls

 

Complex Containers

Canvas

Use this as your base class if you are designing your own Control as it provides partial support for a scrollable control.

Frame

In order for any control to be visible it must be placed in a Frame. A Frame can be complex, having a draggable title bar, close button and resize bar, or it can be simple and borderless displaying its contents at a fixed location. You will almost never have to create a Frame yourself. You would usually use a Form to display controls on-screen. The Form object will create its own Frame when you request it to be displayed.

Window

This is represents the OS native window or the application drawing surface (for OSes that do not support multiple windows for an application). Again, you would almost never have to create a Window yourself, a Form object is capable of creating and displaying a Window to contain its Frame which then contains the Form.

Form

This is a container that is designed to hold a form-like user interface. It is also capable of displaying itself on screen in a new Frame and Window (if necessary) through its execute() or show() methods.

Editor

This is a special type of Form that allows for the automatic transfer of data from an Object field to an on-screen UI control. This is explained in the chapter GUI Data Transfer and Live Objects.

AppForm

This is a special type of Editor designed to be used as the "main" form for an application. It provides a number of features including the automatic setup of toolbars and menu bars as well as being SIP aware on the PocketPC. The use of an AppForm is explained in the chapter Application GUI Issues.

Table Based Controls

TableControl

This is the basic Table control in Ewe and can be used to provide very powerful data presentations or useful more complex controls.

TreeControl

This is actually a type of TableControl.

 

Using Table and Tree controls are explained in their own chapters in this guide.

Mosaic Based Controls

These controls are in the ewe.graphics package and are explained in detail in the chapter on Mosaic Controls.

Mosaic

This is control is used to display, flicker free, any number of images on screen, some or all of which may be moving or be animated.

MosaicPanel

This is an extension of Mosaic and provides methods that you override to detect when the user moves the mouse around the Mosaic, over and off of images, or when the user presses the pen/mouse on the images.

InteractivePanel

This is an extension of MosaicPanel and allows selected images to be dragged around the MosaicPanel and dropped in any location or on each other. Methods are provided to detect when images are picked up, dropped or moved over each other. The Solitaire example uses an InteractivePanel for its implementation.

FileChooser Controls

These controls allow you to browse and select files on a file system or virtual file system. They are located in the ewe.filechooser package.

FileChooser

This is the standard FileChooser and can be the base for developing more complex FileChoosers.

ImageFileChooser

This is a FileChooser that displays an image preview of decodable images on the file system.

Miscellaneous Controls

ImageControl

This is used to display an Image in a container. The displayed image can be changed and you have options for scaling the image.

PageDisplay

This is in the ewe.graphics package. It provides a way of displaying a set of displayable units (PageDisplayUnit objects) in a scrollable canvas. However, it is much simpler than a Mosiac and does not clip the displayed units like a normal container. It can be used as the base for a control like a web browser, for example.
