The Mosaic Control

The ewe.graphics.Mosaic Control, when used with ewe.graphics.AniImage images provides a very easy way to write 2D graphics games and applications. The Mosaic takes care of displaying multiple moving/animated images without flicker or refresh problems. There are three levels of Mosaics which you can use:

ü      Mosaic  this is the bottom level control. It handles the screen display and refreshing individual images or screen areas.

ü      MosaicPanel  this control adds pen interpretation code. It decodes pen presses to determine if any image is being pressed, clicked or dragged.

ü      Interactive Panel  this control provides most of the functionality of an interactive scene. It allows images to be picked up, dragged and dropped. It catches circumstances where one image is dragged over or off another, or when one image is dropped on top of another.

Mosaic

Using a Mosaic is fairly easy. After you create the Mosaic, you create AniImage objects and add them to the Mosaic using addImage(). You can remove images by using removeImage(). Use the move(int x, int y) or translate(int dx, int dy) methods to move the images within the Mosaic and use one of the change() methods to change the appearance of the image. After moving/changing the image, call refresh() on the AniImage to update it on the screen. You can also call refresh(AniImage which,Graphics g) on the Mosaic, but the first method is the preferred way. Use refresh() on the Mosaic to refresh the entire Mosaic.

 

These are the main methods you need to create an animated scene or game  and if you do not need to respond to pen presses, this is probably all you would need to use.

MosaicPanel

Use this when you want to respond to pen input. It provides a set of methods that indicate when the user has pressed/released/dragged an image in the Mosaic. Override these methods to provide your functionality. These methods include: imagePressed(), imageReleased(), imageBeginDragged(), imageDragged(), imageNotDragged() and some others.

 

Note that this control does not implement any image moving  if you need to have images moved with pen input then use an InteractivePanel.

InteractivePanel

This panel catches pen inputs and implements the moving of images for you. It provides an additional set of methods to indicate when images are dragged/dropped over other images. In order for this to work correctly you must note the following:

 

ü      If an image is to be moved on screen using the pen, it must have the IsMoveable bit set in the properties variable.

ü      If you want to detect if an image is dragged over a destination image you must add the destination image to the touching Vector of the InteractivePanel. Only these images are checked against moving images to see if the moving image is either touching it or is being dragged over/off it.

Mosaic Sample

The file samples\mosaic\MosaicDemo.java has an example of the use of an InteractivePanel. It shows a folder and five pages. The pages can be picked up and dropped into the folder  the folder itself is also moveable. When all pages have been dropped in the folder a message box will be displayed.
