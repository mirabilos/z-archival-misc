Reflection API

Like Java, Ewe has a number of features that allow the programmer to discover, at runtime, information about a particular class. Although a lot of the Ewe reflection API is quite different to the Java reflection API, it provides all the same functionality. The Ewe API achieves the same functionality using much fewer classes and methods.

Java Type Encoding

The ewe.reflect.Reflect class is used to provide a number of reflection methods. Many of these methods require the specification of Java types (both primitive and Object types) as Strings, or they return Java types as Strings. The convention used by Java to encode types is described below. These are referred to as java encoded types.

java.lang.Class

This class in Ewe implements a large subset of its functionality in Java. The Ewe API for Class specifies what it actually does provide. The functionality in Class that is not provided by Ewe relates to the discovery of Methods, Constructors and Fields. The ewe.reflect.Reflect class is used for this and will be discussed later.

Class.forName() vs Reflect.forName()

A Class object can represent the class of any Object, Array or Primitive type. The static Class.forName() method is normally used in Java to acquire a Class object that represents a particular Class, given the full package name (not Java type encoded). However, under the Java VM this method is unable to load classes that are stored in a .ewe file. Therefore the static Reflect.getForName() method should be used instead. This provides a new Reflect object that represents the specified class, and you can then use the Reflect.getReflectedClass() method to retrieve a Class object that also represents that class.

 

Another useful method in Reflect is Reflect.getClass(String type, Class requestor). This is the most flexible method, allowing you to get classes representing primitive types as well. It therefore requires a java encoded type name. The requestor argument is optional (may be null) and specifies the Class that is requesting the getClass() method. If the class could not be retrieved via a normal lookup, then the ClassLoader of the requestor class (if any) is also checked to see if it could the class.

ewe.reflect.Array

This class has a number of static methods that provide information about any array object. It has a number of methods in common with the standard java.lang.reflect.Array class including:

ü      boolean isArray(Object array) – Returns true if an Object is an array.

ü      Class getComponentType() – Get the type of the individual elements of the array.

ü      int getLength() – Get the length of the array.

ü      Object newInstance(Class componentType,int length) – Create a new array.

 

Array also provides a method to set the value of an array element and one to retrieve the value of an array element. These are:

ü      Wrapper getElement(Object array,int index,Wrapper dest)

ü      void setElement(Object array,int index,Wrapper value)

 

These two methods utilize a ewe.reflect.Wrapper object to transfer any Java value. This includes primitive types and object/array types. Wrapper objects are also used when invoking methods and setting/getting field values through reflection.

 

Note: Java uses different wrapper Objects for different data types (e.g. Integer, Character, etc.). However, to reduce the size of the Ewe library, only the Wrapper class is used for all data types.

ewe.reflect.Wrapper

The Wrapper class is very easy to use and a Wrapper object is reusable.  To set the value of a Wrapper (and therefore also set the type of the Wrapper) you would use one of the set<type>() methods. For example, calling wrapper.setInt(1000); will set the type of the Wrapper to Wrapper.INT and also set its value to be 1000.

 

The get<type>() methods are used to retrieve the value within a wrapper that has been set by the VM or by a set<type>() method. However if you call one of the get methods that is not exactly the correct type of the Wrapper (e.g. trying to call getInt() when the Wrapper type is Object) will cause a RuntimeException to be thrown.

 

The to<type>() methods are used to convert the data stored within a Wrapper to a particular type. This is only allowed if the types are considered compatible by the VM. For example you can call toInt() if the type of the Wrapper is INT, CHAR or SHORT, but not if it is DOUBLE or OBJECT.

 

You can use getType() to discover the type of the Wrapper.

ewe.reflect.Reflect

This class does most of the work of Reflection. It existed before the Class and Array classes were implemented under Ewe and so has a number of methods that perform the same function as those that exist under Class and Array.  However you should try to use the Class and Array methods in preference to the Reflect methods.

 

Because Reflect existed before Class, most of the methods use java encoded type Strings instead of Class objects to specify data types. This also tends to be a little faster since new Class objects need not be created during method calls.

 

The task that you will most likely use Reflect for is to acquire Method, Constructor and Field information on a particular Object. This is done using the getMethod(), getConstructor() and getField() methods. These methods, as well as the methods for using these Objects are documented in the API.
