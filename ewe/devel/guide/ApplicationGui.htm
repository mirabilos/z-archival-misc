Application GUI Issues

So far we have dealt with Ewe's GUI from the standpoint of creating general Forms. This chapter deals with more detailed issues especially those related to the creation and display of your application's Main Form.

 

This chapter will be of particular interest to mobile developers, mostly because of the variations of displays on the various mobile devices that their applications may target. For example, the PocketPC platform provides a button to activate and select the type of input panel to be used. This button is at the bottom right of the screen, within the user's application screen space, and may be programmed to appear and disappear. It may be the case that you wish to display data or tool/menu bars along this bottom row but do not wish to have it be obscured by this button. Ewe provides the necessary tools to do this.

 

Ewe provides a SipButton control to handle the possible existence of a SIP button, which effectively acts as a placeholder, reserving space where the SipButton would be. However the Form control provides methods for creating toolbars and other controls that will automatically use this control. As a result you should not need to use this control yourself when building your forms.

 

Additionally, most GUI elements are designed with space saving in mind. For example the mTabbedPanel, very useful for your application's main Form - especially on PDAs with limited screen space, can allow you to place controls in line with its tabs, either to the right or left of them.

Simple Form Enhancement Methods

These methods are useful for adding controls to a Form that are SIP aware and will look good on any platform. Generally, you should call these methods first before adding any other controls, since they generally return specific CellPanels that you should then use to add your controls.

public CellPanel [] addToolbar()

This method will add a horizontal CellPanel to the Form. The location where the toolbar is added depends on the running OS. On PocketPC it is at the bottom, on others it is generally at the top. If the OS is PocketPC, then space is reserved for the SIP button within the toolbar.

 

This method sets up the Form as necessary and then returns an array of two CellPanels. The CellPanel at index 0 is the toolbar panel to which you should add your toolbar controls. The CellPanel at index 1 takes up the rest of the Form and is the one that you should add your Form's main data controls.

 

Here is a simple example, it adds a toolbar and then adds a MenuBar and tool buttons to the toolbar. Note that this control inherits from Editor which is a type of Form.

 

package samples.ui;
import ewe.ui.*;
import ewe.fx.*;
 
//##################################################################
public class TestForm extends Editor{
//##################################################################
 
//===================================================================
public TestForm()
//===================================================================
{
        title = "Test Toolbars";
        CellPanel [] p = addToolbar();
        p[0].defaultTags.set(INSETS,new Insets(0,1,0,1));
        //
        MenuBar mb = new MenuBar();
        Menu m = new Menu(new String[]{"Open","Save","-","Quit"},"File");
        mb.addMenu(m,"File");
        p[0].addNext(mb).setCell(DONTSTRETCH);
        //
        // Add a toolbutton.
        //
        Control c = new mButton(new mImage("ewe/savesmall.bmp",Color.White));
        c.setToolTip("Save Document");
        p[0].addNext(c).setCell(DONTSTRETCH);
        c = new mButton(new mImage("ewe/exitsmall.bmp",Color.White));
        c.setToolTip("Quit");
        p[0].addNext(c).setCell(DONTSTRETCH);
        //
        // Add main data control, a mTextPad in this case.
        //
        p[1].addLast(new mTextPad(20,40));
}
 
//##################################################################
}
//##################################################################

Handling events generated from these controls is as simple as adding EventListeners to the toolbar container (p[0] in our example).

public Control [] addTabbedPanel(boolean useExpandingTabs)

Tabbed panels are provided in Ewe by the mTabbedPanel class. There are two main purposes for using these.

 

The first and more common purpose is to split up your user interface into different tabbed sections. In this case you would set the useExpandingTabs parameter to false. This method would then add a tabbed panel to the Form that would take up the entire area of the Form. The tabs would be placed either at the top of the tabbed panel, or at the bottom depending on the OS. Under PocketPC, the tabs are placed at the bottom, in line with the SIP button. With this parameter, the returned Control array contains only one element, which is the mTabbedPanel object itself.

 

The second purpose, which is something unique to Ewe, is to provide multiple toolbars when there is limited screen space (as on a PDA). This is made possible via the auto-expanding feature of the mTabbedPanel class. This feature allows the data area of the tabbed panel to appear when a tab is selected, and completely disappear when no tabs are selected. With this feature, you generally also have a data area where your application's main data is shared. As the toolbar tabs expand and contract, the main data area is resized appropriately. To select this feature you would set the useExpandingTabs parameter to true. This method would then return an array of two elements. The element at index 0 would be the added mTabbedPanel, and the one at index 1 would be the CellPanel into which you should place your main application data.

 

The example below shows how you use addTabbedPanel() with the expanding tabbed panel option.

 

package samples.ui;
import ewe.ui.*;
import ewe.fx.*;
 
//##################################################################
public class TestTabForm extends Editor{
//##################################################################
 
public MenuItem menuBar = new MenuItem();
 
//===================================================================
public TestTabForm()
//===================================================================
{
        title = "Test Tabs";
        //
        Control [] p = addTabbedPanel(true);
        //
        mTabbedPanel mt = (mTabbedPanel)p[0];
        CellPanel data = (CellPanel)p[1];
        //
        CellPanel fileTools = new CellPanel();
        Card c = mt.addCard(fileTools,"File",null);
        c.iconize(ewe.io.File.getIcon(ewe.io.File.ClosedFolderIcon),true);
        fileTools.defaultTags.set(INSETS,new Insets(0,1,0,1));
        fileTools.addNext(new mButton(new mImage("ewe/opensmall.bmp",Color.White))).setCell(DONTSTRETCH);
        fileTools.addNext(new mButton(new mImage("ewe/savesmall.bmp",Color.White))).setCell(DONTSTRETCH);
        fileTools.addNext(new mButton(new mImage("ewe/exitsmall.bmp",Color.White))).setCell(DONTSTRETCH);
        //
        CellPanel editTools = new CellPanel();
        c = mt.addCard(editTools,"Edit",null);
        c.iconize("ewe/editsmall.bmp",Color.White,true);
        editTools.defaultTags.set(INSETS,new Insets(0,1,0,1));
        editTools.addNext(new mButton(new mImage("ewe/copysmall.bmp",Color.White))).setCell(DONTSTRETCH);
        editTools.addNext(new mButton(new mImage("ewe/cutsmall.bmp",Color.White))).setCell(DONTSTRETCH);
        editTools.addNext(new mButton(new mImage("ewe/pastesmall.bmp",Color.White))).setCell(DONTSTRETCH);
        //
        data.addLast(new mTextPad(20,40));
}
 
//##################################################################
}
//##################################################################

Modifying Tab Behavior

The addCard() method of mTabbedPanel is used to add a new tab to the panel and returns a Card object. There are a number of things you can do with this object, but the one that is of most interest is to call one of the iconize() methods. This method associates an icon with the tab that is used along with the name of the tab as specified by addCard(). The iconize() method comes in a number of forms allowing  you to specify the icon in a number of ways. Some of these methods have a parameter called iconOnlyWhenClosed. If this parameter is set true (as it is in the above example) then when that tab is not selected, only the icon will be displayed, and not the text of the tab name.

 

In cases where the iconOnlyWhenClosed is set true for a card, it is possible to also tell the mTabbedPanel to never show the text of the tab, but to only show the icon even when the tab is selected. This may be desirable for applications like the one above, where you are using the mTabbedPanel as a container for tool buttons. If you want this to be the case, then you should set the dontExpandTabs member of mTabbedPanel to be true.

Adding Controls Next To Tabs

This is another feature unique to Ewe and very useful for small screen displays. It allows you to place controls to the left and right of the tabs. Because of the position of the SIP button, it is advised that you add controls to the left of the tabs only, but this is completely up to you.

 

To do this call the getExtraControls(boolean isLeft) method of the mTabbedPanel. This returns a CellPanel that will be placed to the left of the tabs (isLeft == true) or the right of the tabs (isLeft == false). For example if you add the following lines of code to the example:

 

        MenuBar mb = new MenuBar();
        Menu m = new Menu(new String[]{"Open","Save","-","Quit"},"File");
        mb.addMenu(m,"File");
        mt.getExtraControls(true).addNext(mb).setCell(DONTSTRETCH);

The AppForm Class

This class, which inherits from Editor, uses all the features described above and provides convenient variables to which you can add your menus, tools and data controls. The various constructors for the AppForm will create either a toolbar, or a tabbed panel for your Form. If you choose to use a tabbed panel, you can additionally choose to use an auto-expanding tabbed panel or not.

 

Here are the constructors for AppForm() 

 

Constructor Summary

AppForm()
          Create a new AppForm() with no tabbed panels.

 

AppForm(boolean useTabs, boolean autoExpandingTabs)
           Create a new AppForm() that can be set to use tabbed panels or not.

 

AppForm(Object objectToEdit)
           Create a new AppForm() that does not use tabbed panels and will also edit the specified object.

 

AppForm(Object objectToEdit, boolean useTabs, boolean autoExpandingTabs)
          Create an AppForm that can optionally use tabs and will also edit the specified object.

 

AppForm(Object objectToEdit, boolean useTabs, boolean autoExpandingTabs, boolean placeMenuAndTools)
          Create an AppForm that can optionally use tabs and will also edit the specified object, and can optionally not place the menu and tools.

 

 

 

The second constructor is the one that you would most likely use. If useTabs is true then a tabbed panel will be used and you can optionally specify to use auto-expanding tabbed panels.

 

After the constructor you should add your menus to the menus member variable (a MenuBar), add your tools to the tools member variable (a CellPanel), add tabs to the tabs member variable, and data to the data member variable.

 

Here is a quick example:

 

package samples.ui;
import ewe.ui.*;
import ewe.fx.*;
 
//##################################################################
public class TestForm extends AppForm{
//##################################################################
 
//===================================================================
public TestForm()
//===================================================================
{
        // Going to use expanding tabs.
        super(true,true);
        //
        title = "Test AppForm";
        //
        Menu m = new Menu(new String[]{"Open","Save","-","Quit"},"File");
        menus.addMenu(m,"File");
        //
        CellPanel fileTools = new CellPanel();
        Card c = tabs.addCard(fileTools,"File",null);
        c.iconize(ewe.io.File.getIcon(ewe.io.File.ClosedFolderIcon),true);
        fileTools.defaultTags.set(INSETS,new Insets(0,1,0,1));
        fileTools.addNext(new mButton(new mImage("ewe/opensmall.bmp",Color.White))).setCell(DONTSTRETCH);
        fileTools.addNext(new mButton(new mImage("ewe/savesmall.bmp",Color.White))).setCell(DONTSTRETCH);
        fileTools.addNext(new mButton(new mImage("ewe/exitsmall.bmp",Color.White))).setCell(DONTSTRETCH);
        //
        CellPanel editTools = new CellPanel();
        c = tabs.addCard(editTools,"Edit",null);
        c.iconize("ewe/editsmall.bmp",Color.White,true);
        editTools.defaultTags.set(INSETS,new Insets(0,1,0,1));
        editTools.addNext(new mButton(new mImage("ewe/copysmall.bmp",Color.White))).setCell(DONTSTRETCH);
        editTools.addNext(new mButton(new mImage("ewe/cutsmall.bmp",Color.White))).setCell(DONTSTRETCH);
        editTools.addNext(new mButton(new mImage("ewe/pastesmall.bmp",Color.White))).setCell(DONTSTRETCH);
        //
        tools.addNext(new mButton(new mImage("ewe/exitsmall.bmp",Color.White))).setCell(DONTSTRETCH);
        data.addLast(new mTextPad(20,40));
}
//##################################################################
}
//##################################################################
