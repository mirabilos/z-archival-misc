Important GUI Issues

This chapter deals with a number of issues having to do with the Ewe GUI library.

Determining Screen Characteristics

How you decide to layout your controls at run-time may well be determined by the screen dimensions or the type of system the application is being run on. You can determine a number of parameters about the running system using the following methods.

 

public static boolean ewe.ui.Gui.screenIs(int flags)

 

This method returns true if the Gui system considers the screen to have the characteristic as specified by the flags parameter. This parameter can have one of the following values:

  • BIG_SCREEN  screen width >= 600 and height >= 300.
  • WIDE_SCREEN  screen width >= 600 and width > height.
  • LONG_SCREEN  screen height >= 290 and height > width.
  • DESKTOP_SCREEN  screen width >= 640 and height >= 480.
  • DESKTOP_WIDTH  screen width >= 640.
  • DESKTOP_HEIGHT  screen height >= 480.
  • PDA_SCREEN  screen width <= 350 and screen height <= 350.

 

If you need to determine the actual size of the screen then you can use the line:

 

Rect screenSize = (Rect)Window.getGuiInfo(Window.INFO_SCREEN_RECT,null,new Rect(),0);

 

You can also determine particular characteristics of the GUI system by using the method:

 

public static int ewe.ui.Gui.getGuiFlags();

 

This returns an integer value containing a set of flags. These flags will be the bitwise OR of the following:

 

·        Window.GUI_FLAG_HAS_TASKBAR  which indicates that the OS has a Taskbar in which icons can be placed (e.g. Windows).

·        Window.GUI_FLAG_REVERSE_OK_CANCEL  which indicates for this particular device you should place Cancel buttons before OK (normally it is the other way around) because that is how it is laid out on the device (e.g. on the Sharp Zaurus).

 

You can also determine if the application is running on a system that is considered to be mobile by using

 

public static boolean ewe.sys.Vm.isMobile();

Images in the Ewe Library

There are a number of images in the Ewe library that you can use for your applications. They are generally used to create an mImage object with a transparent mask or transparent color. For example:

 

mImage mi = new mImage("ewe/bottomarrowsmall.bmp", Color.White);

or

mImage mi = new mImage("ewe/imagesmall.bmp", new Color.(0, 255, 0));

or

mImage mi = new mImage("ewe/Page.bmp", "ewe/PageMask.bmp");

 

These can then be used as icons for a variety of controls.

Some Important Constants

These are values that are returned by standard Forms such as MessageBox:

 

Name

Returned when

IDCANCEL

Cancel or Window 'x' button pressed.

IDOK

OK button pressed.

IDYES

YES button pressed.

IDNO

NO button pressed

IDBACK

BACK button pressed.

 

These are values that can be passed as a type of MessageBox:

 

Name

Displays

MBOK

A single OK button.

MBOKCANCEL

OK and Cancel buttons.

MBYESNO

Yes and No buttons.

MBYESNOCANCEL

Yes, No and Cancel buttons.

Some Important Methods

static void Gui.takeFocus(Control c, int how);

Switch focus to a particular control. See the API to see the possible values of how.

 

static void Gui.setOKCancel(Form f);

Puts the OK and Cancel buttons in the appropriate place in a Form (either at the bottom or in the title).

 

static void Gui.setStyle(int style);

Sets the style to be STYLE_ETCHED or STYLE_3D or STYLE_SOFT (STYLE_SOFT is not recommended for mobile devices due to slower display times).

Some Important Classes

 

ewe.fx.TextAndIcon

This is used to display an icon and text together as a single image. This is useful since most controls can either display a line of text or an icon. Most of the times this will be done automatically for you and so you do not need to create one explicitly, but it may be useful in certain circumstances.

ewe.fx.ImageCache

This is used to fetch images that you may use more than once. Rather than have multiple copies of the image this will get the image once, and any requests for the same image will return the previously fetched image. To get an image use:

 

IImage get(String imageName, Object maskOrColor);

 

The static cache ImageCache.cache can be used.

ewe.fx.DrawnIcon

This class is an image that is drawn rather than being a bitmapped image. There are two DrawnIcon types defined: TICK and CROSS. They pen that is used to draw these images can be specified in the constructor.
