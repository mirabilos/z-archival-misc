Quick-Start Issues

This chapter deals with some issues that will be important to experienced Java programmers who wish to get working on Ewe as quickly as possible. Since the Ewe VM can execute applications from separate class files, just like a Java VM, the use of Jewel during initial program development is not necessary. Jewel is really used for packaging your application for deployment. Hence programmers can start writing and testing programs without reading the next chapter on Jewel.

The main() method and console output

You can start your Ewe application using a static void main(String []args) method as in standard Java, however you must begin and end the method with two special function calls.

 

import ewe.sys.*;
//==============================
public class HelloWorld {
//==============================
 
public static void main(String args[])
{
        Vm.startEwe(args);
        Vm.debug("Hello World!");
        try{
               mThread.sleep(2000);
        }catch(InterruptedException e){}
        Vm.exit(0);
 }
 
//==============================
}
//==============================
 

As you can see, the first line of the main() method should be Vm.startEwe(String args[]). This starts and initializes the Ewe library. This is only necessary if you are using the main() method for starting your Ewe application. It is not necessary if you are starting your Ewe application by overriding the mApp class or any of the other allowed starting classes. You should also use Vm.exit(int code) to exit your Ewe application.

 

You will also see that to output a line of text to the console, you should use the Vm.debug(String message) method. Note that this method has no effect when run on a mobile system that does not support consoles (e.g. WindowCE/PocketPC).

Multi-threading and Synchronization

The ewe.sys.mThread class

In the example above you will see a call to mThread.sleep(). Ewe does not support true pre-emptive multi-threading, and so the java.lang.Thread class is not provided. However it does support a version of co-operative multi-threading (where each thread yields its processor time to other threads via the sleep() method) provided by the ewe.sys.Coroutine class. Threads also yield to other threads during blocking IO calls and during ewe.sys.Handle wait() method calls – so for the most part, you can have a fully multi-threaded application and never have to call sleep().

 

The ewe.sys.mThread class provides an API that is almost identical to java.lang.Thread except that it uses Coroutine threads instead of pre-emptive Threads. You should use mThread objects instead of Coroutine objects for writing multi-threaded applications – since it provides a much more familiar API.

 

Note that the main() method is run within an mThread.

 

For example the following code will work under Java. It is an example of a Thread that prints out a message once a second.

 

class ATestThread extends Thread // Use ewe.sys.mThread for a Ewe application.
{
     int myID;
 
public ATestThread(int myID)
{
     this.myID = myID;
}
public void run()
{
     while(true){
             System.out.println("Tick - "+myID); // Use ewe.sys.Vm.debug() for a Ewe application.
             try{
                     sleep(1000);
             }catch(InterruptedException e){}
     }
}
}

 

You could start this thread by calling new ATestThread(1).start().

 

This same code will also work under Ewe by simply substituting ewe.sys.mThread instead of Thread, and substituting ewe.sys.Vm.debug() instead of System.out.println()

mThread Synchronization

Since Ewe does not support true threads, the wait() and notify() methods of the Object class and the synchronized keyword should not be used. Instead, a very similar synchronization scheme is used that uses ewe.sys.Lock objects. This works as follows:

 

1. Synchronization - Instead of:

synchronized(any_object){
  //Do some code.
}

Use:

a_lock_object.synchronize(); try{
  //Do some code.
}finally{a_lock_object.release();}

 

 

2. Waiting - Instead of:

     try{
any_object.wait(1000);
     }catch(InterruptedException e){}

Use:

     try{
             a_lock_object.waitOn(1000);
     }catch(InterruptedException e){}

 

3. Notifying – Instead of:

     any_object.notify();
     any_object.notifyAll();

Use:

     a_lock_object.notifyWaiting();
     a_lock_object.notifyAllWaiting();
 

Note that these operations can only be done on ewe.sys.Lock objects and the first two can only be done within a running mThread.
