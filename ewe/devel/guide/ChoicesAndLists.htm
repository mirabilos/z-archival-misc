Choices, Menus and Lists

The Ewe UI library comes with a number of related choice, menu and list controls, each of which are dynamic (the items they display can be modified dynamically).

ewe.ui.ChoiceControl

This is the base for all choice controls and so its methods apply to all these controls. The items in the choice control are stored in a publicly accessible Vector called items. You are free to view and modify this Vector although you are advised to use one of the different methods that add/remove/list the items instead.

 

After adding/removing items to your ChoiceControl you should call updateItems(). This tells the control to update itself and the scroll bars of its ScrollServer to reflect the new items.

 

While most choice controls use this Vector for item storage, the class SimpleList does not. That class requires you to override specific methods to dynamically fetch the items in the list. This allows you to display arbitrarily large lists without having to store data for each item in memory since you can fetch or create each item as it is needed.

 

Each item in a list can be either a simple String or a MenuItem. A MenuItem can have sub-menus, icons, be disabled or enabled or even be indented.
