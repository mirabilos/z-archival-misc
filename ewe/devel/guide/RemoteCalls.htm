Ewe RMI - Remote Calls

In this chapter we will look at the Ewe Remote Call mechanism. It can be used as a very quick and easy way for inter-process communication for client-server applications or for data synchronization. It also allows for fully secure communication using Ewe's security package.

 

Usually you would first establish a communication path between the client and the server using some sort of Stream (e.g. a Socket). Then a RemoteCallHandler will be created on each end of the communication stream. This object is responsible for sending, decoding and invoking remote calls on remote objects and also for packaging and sending back results.

 

To simplify these operations, a RemoteCallTask is an object that you can easily extend to provide a RMI environment for your application.

The ewe.io.RemoteCall Class

 

The RemoteCall object is used as follows:

  1. The RemoteCall is created given the specifications of a method on a remote object.

  2. Parameters are added to the RemoteCall one at a time.

  3. The RemoteCall is invoked via a RemoteCallHandler object.

  4. The RemoteCallHandler sends the call and then waits for a reply.

  5. The reply is passed back to the RemoteCall which provides the return value if present.

 

The RemoteCall is fully asynchronous and is in fact a type of Handle. After invoking the call you can check on its progress at any time or wait for the call to be complete for a specific length of time. You can also call it synchronously, waiting until the call is complete and a reply has been received. You can set a timeout before making the call.
