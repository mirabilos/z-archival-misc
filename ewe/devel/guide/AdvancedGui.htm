Advanced GUI Features

This chapter deals with the use of some of the more advanced GUI features of Ewe, including Tool Tips, Hot Keys and Context Menus.

Tool Tips

Tool tips are very easy to use in Ewe. The simplest way to assign a tip to a Control is to simply call the setToolTip(Object tip) method. The tool tip can be one of the following classes:

 

ü      A String. This can span multiple lines – simply separate each line with a '\n' character.

ü      An object that implements ewe.fx.IImage. This will be displayed within the tool tip box.

ü      A Control. The control will be drawn in the tool tip box, but will not be active.

ü      A ToolTip object. This has a field called tip which should be of one of the three types above, and a boolean field called persists. If persists is true then the tip stays on screen until the mouse exits the associated Control completely.

Providing a Location Sensitive Tool Tip

Calling setToolTip() will automatically request PenMoved events for the control and will set up the control for tool tip display. When the GUI system decides that a tool tip should be displayed (when the mouse pointer hovers over the control) it will call Object getToolTip(int x, int y). This prompts the control to return the appropriate tool tip for the location as specified by the x and y co-ordinates (which will be relative to the top left of the control). By default this simply returns whatever tip was used in setToolTip() (which may be null) but you can override this to provide a different tool tip depending on the value of x and y.

Hot Keys

Most controls can have hot keys assigned to them to allow for direct activation or focusing of the control. These hot keys can be a combination of the CTRL, SHIFT and a letter/number/function key.

 

The setHotKey(int modifier, char key) or setHotKey(int modifier, int keyCode) method can be used to set the hot key for a control. The modifier can be a combination of IKeys.SHIFT and IKeys.CONTROL or it can be IKeys.ONLY  or 0. A value of 0 indicates that the key pressed in combination with either SHIFT or CTRL or both should be recognized as a hot key. A value of IKeys.ONLY indicates that the specified key must be pressed alone in order to be recognized as the hot key.

Specifying a Hot Key in a Text Label

Most controls which display text as a prompt (e.g. mButton, mCheckbox, mLabel, etc.) can have their hot key set automatically by providing a specially formatted text label. This is done by appending a $ followed by a single character to the end of the text for the control. That single character will be taken to be the hot key for the control and the first instance of that character will be underlined when the text is displayed on the screen.

 

For example you could do:

 

mButton mb = new mButton("Stop$s");

mLabel label = new mLabel("First Name:$f");

Context Menus

Most controls will display a context-sensitive popup menu when the right mouse button is clicked or a tap and hold gesture is done. You can set the menu that is displayed by using the setMenu(Menu m) method in Control (if you set the parameter to null, then no menu will be displayed). You can handle the item selected by the user by overriding the popupMenuEvent(MenuEvent ev)  or the popupMenuEvent(Object selectedItem) which is called from the first method. The first method is called if the user selects an item or cancels the menu, while the second method is only called if the user selects an item.

Creating a Menu Dynamically

The doShowMenu(Point pen) method is called when the Control wants to display a popup-menu. You can override this method to provide a different menu to display depending on the pen location. You should create the menu, call setMenu() and then call super.doShowMenu(pen). Note that if the menu was requested to be displayed programattically or via a key-press, then the pen parameter will be (0,0).

Merging with the Clipboard Menu

The method Menu getClipboardMenu(Menu addTo) can be used to add the clipboard menu to an existing menu (the addTo parameter). If addTo is null then a clipboard menu alone will be returned.

 

The method checkClipboardOperations(Menu m) is used to disable/enable clipboard items depending on the editable state of the Control and whether or not there is data in the clipboard. Before a popup menu is displayed, the boolean checkMenu(Menu m) method is called. You can call the checkClipboardOperations() method in this method.
