More I/O Topics

 

The ewe.io.DataProcessor Interface

This provides an API for a general data processing operation where an input sequence of bytes is transformed into a processed output sequence of bytes. This output sequence may not necessarily be of the same length as the input sequence. This interface can therefore be applied to a variety of processes such as compression/decompression and encryption/decryption. The interface members are listed below:

 

int getBlockSize()  Some data processors require each block to be processed to be multiples of a certain number of bytes (e.g. DES encryption requires blocks which are multiples of 8-bytes). The method is used to query the processor to find out what this value is. If the method returns a value of 1 or less, then it can be assumed that there is no block size  i.e. each block to be processed can be of arbitrary size.

 

int getMaxBlockSize()  Some data processor may have a maximum input block size that they can handle at a time. If this method returns a value of zero or less, then input blocks can be of arbitrary length.

 

void closeProcessing()  This destroys all resources used by the DataProcessor. No other methods should be called after this method is called.

 

ByteArray processBlock(byte [] source, int offset, int length, boolean isLast, ByteArray destination)  This sends a block of data to the data processor and accepts processed output. Note that the number of bytes which are output may not necessarily be the same amount as is input. The processor may require more input to produce some output or it may produce output which is larger than the input. Any processed data which is ready for output is placed in the provided destination ByteArray and is returned. If no destination ByteArray is provided, a new one will be created and returned. If there is an error in the processing, this method will return null.

The isLast parameter is used to tell the processor that there will be no more input blocks of data after this one. It is acceptable to send a block of data of zero length with isLast being true. If this parameter is true the DataProcessor should:

 

1.       Accept any input data provided with the method call and then process and output all pending unprocessed data.

2.       Reset itself to its initial state such that the next call of processBlock() will be considered to be the beginning of a fresh sequence of data.

 

There is a static method in ewe.io.IO that will process all the bytes in a sequence of bytes using a DataProcessor.

 

static ByteArray processAll(DataProcessor dp,byte [] source,int offset,int length,ByteArray dest) throws IOException

 

This method deals with padding the data if it is not a multiple of the block size of the DataProcessor.

 

Other DataProcessor Issues

The Base64Encoder and Base64Decoder are DataProcessors that can be used to convert bytes into Base64 ASCII text and vice-versa. This type of encoding is commonly used to attach binary documents to other documents which are usually encoded using 7-bit ASCII (e.g. standard Internet e-mails). The encoder will convert every sequence of 3 data bytes into a sequence of 4 ASCII bytes, up to a maximum number of bytes per line, after which a CR\LF is inserted. Pad bytes are added at the end if the number of input bytes is not a multiple of 3.

 

The DataProcessingStream is used to process data either before it is written to an output Stream or after it is read in from an input Stream.

The ewe.io.TextCodec Interface

This method is used to translate sequences of bytes into sequences of Unicode characters and vice-versa.

 

ByteArray encodeText(char[] text, int start, int length, boolean endOfData, ByteArray destination) throws IOException;

 

This method converts text characters into a byte sequence and places it in the destination ByteArray (or a new one if it is null). The endOfData parameter is used to indicate the end of data and it should therefore process and pending data and return all processed data.

 

CharArray decodeText(byte[] bytes, int start, int length, boolean endOfData, CharArray destination) throws IOException;

 

This performs the reverse of encodeText(), it converts the encoded byte sequence into a sequence of chars and places it in the destination CharArray.

 

Additionally there are two more methods:

 

void closeCodec() throws IOException;

Object getCopy();

 

The closeCodec() method aborts any encoding and the codec should not be used afterwards. The getCopy() method is used to get a new instance of the codec, which is ready to encode/decode a fresh sequence of bytes/chars.

 

TextCodec objects are used by the InputStreamReader class to convert bytes read by the InputStream into characters read using the read() methods. You can set the codec explicitly by modifying the codec variable, or it can be set upon creation by specifying the codec name.

 

The codec names which are provided in the Ewe library are: ewe.io.ASCII_CODEC (which is the constant String "ASCII") and ewe.io.JAVA_UTF8_CODEC (which is the constant String "JAVA_UTF8")  and "UTF-8" which is the same as "JAVA_UTF8".

 

You can implement and add more codecs by using the ewe.io.IO static methods:

 

static void setCodec(String codecName, TextCodec codec);

 

You can get a named codec (if present) using:

 

static TextCodec getCodec(String codecName);
