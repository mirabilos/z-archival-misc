Remote Connection

Ewe provides a simple mechanism for allowing a mobile device to communicate with a PC to which it is attached. A connection between the two systems is provided by small server processes that run on both devices. These server processes will be OS specific and provide a streaming two-way communication link between the devices. These processes also open up OS independent local TCP/IP ports and the OS independent ewe.io.RemoteConnection object then communicates with the server process through these ports. This architecture ensures portability across different mixes of systems.

 

Under Win32, on the PC side the ewesync.exe program is automatically run when ActiveSync is started. This then activates ewesync.dll on the WinCE device and the two make a communication link that will last until it is stopped or the link is broken. The ewesync.exe program will place a standard ewe icon in the task bar as long as the connection is active. You can explicitly stop the connection by double-clicking this icon and confirming that you want it to stop running.

 

To connect to a Linux PDA such as the Zaurus, the Zaurus EweSync Server can be started from the Ewe VM launcher. This must stay running and will attempt to connect to the EweSync process on the Zaurus every 5 seconds. On the Zaurus, the EweSync Server must be started and can be left running.

 

Please note that because of the nature of the way the Zaurus EweSync Server must connect to the Zaurus itself, it can take up to 30 seconds before a connection is successfully established.

The ewe.io.RemoteConnection Object

This is the object that provides the remote connection services. To use it you first get an instance of RemoteConnection by calling RemoteConnection.getConnection(). This throws an IOException if

a connection could not be made.

 

There are three main types of services provided by the RemoteConnection object.

1. Reading Remote Registry Values

The method String getRegistryValue(String keyAndValueName) can be used to look up a value stored in the registry on the remote device. This does work, even if the remote device is a Linux device, since Ewe simulates a system Registry using a special "Registry.dat" file stored in the same location as the Ewe VM.

 

For example, if you call:

 

String syncValue = rc.getRegistryValue("HKEY_LOCAL_MACHINE\\Software\\EweSoft\\Contact");

 

This will return either the value of the data value named "Contact" in the key "HKEY_LOCAL_MACHINE\Software\Ewesoft" or the value of the default (unnamed) data value of the key "HKEY_LOCAL_MACHINE\Software\Ewesoft\Contact" whichever happens to be present.

2. Connecting to Remote Hosts and Services

You can request a socket connection to a port on a host machine which is reachable from the remote device by calling Socket connectToHost(String hostname, int port). This will return a socket that can then be used to communicate with the specified host on the specified port. On some implementation, this Socket may connect via a proxy within the EweSync server, and on some implementations this may be a direct connection to the host.

 

The problem with this method, however, is that the destination host and port must be known at run time. If you are connecting to the desktop component of an application it is better if the applications do not need to know a precise port number to connect to.  Instead a connection to a named "service" would be more appropriate. A server process on either side of the connection can call: boolean postService(String serviceName, ServerSocket server). Once this is done, an application on the opposite side of the connection can call Socket connectToService(String serviceName). The remote connection object will then be able to look up the service in its list of posted services and provide a connection if possible.

3. Running Remote Applications

You can specify a command line to execute on the remote device (works both ways) by calling:

void execRemote(String commandLine, int execOptions). This will return an IOException only on failure to communicate with the remote host. Returning without a thrown exception indicates that the request was successfully sent to the remote device, but this is no guarantee that the application was actually executed.

 

The operation of this is modified by the execOptions which can be a combination of any of the following values:

 

MOBILE_GET_COMMAND_FROM_REGISTRY – This indicates that the specified command line is actually the name of a registry value that should be looked up on the remote (i.e. a call to getRegistryValue(commandLine) should be made first). The value returned by that call should then be considered the true command line to execute.

 

MOBILE_IS_EWE_APPLICATION – This indicates that the command line (which may have been acquired from the registry) is a Ewe application and the Ewe VM on the remote device should be invoked with the provided command line. This can be used in combination with the previous option.

 

4. Auto-running Applications on Connection

The EweSync server processes can be configured to automatically execute applications when they successfully connect. Your application can register itself on the local machine as such an auto-executing application by calling: RemoteConnection.registerAutoStart(String entryName, String commandLine, int typeAndOptions). The typeAndOptions parameter can be:

 

AUTO_START_IS_EWE_APPLICATION – This indicates that the specified command line is a Ewe application and the Ewe VM should be used to run it.

 

AUTO_START_EMULATOR – This indicates that the command should be executed when the EweSync emulator is connected instead of when the true EweSync connection is made.

 

AUTO_START_SERVICE – This indicates that the command should be executed only when a device on the other end of the connection requests a connection to the named service (which will be the entryName) and that service is not running. The command will be executed and it is expected that the command will set up a server socket and then post the service to the RemoteConnection object. Then the remote application can connect to the service.

 

You can add entries in manually using the EweSync Configuration utility from the Ewe menu in the Ewe launcher.

Bringing It All Together with makeSyncConnection()

In general, data synchronization between a desktop and mobile application requires a process on the desktop side to communicate with a process on the mobile side. Once such a connection is made any type of data exchange is possible. Making this connection reliably is done as follows:

 

  1. The desktop process creates a Server socket to listen to.
  2. The desktop process posts the socket to the RemoteConnection object as a uniquely named (hard coded) service.
  3. The desktop process requests the corresponding process on the mobile device to be executed. It will expect that the mobile process will connect to the named service via the RemoteConnection object on its side.
  4. The mobile process requests a connection to the named service once run.
  5. Once the connection is made both sides will have a Socket with which they can communicate. The Server socket on the desktop side can be closed and data transfer can begin (usually with the desktop process taking charge and controlling the transfer).

 

All these steps are done by a single call to makeSyncConnection() on both the desktop and mobile side (the method operates differently depending on whether the application is running on a mobile device or on the desktop).

 

However before this command will work there are two things that must be done:

 

  1. The mobile application must put in the registry, the correct command line that should be used to run it in synchronization mode.
  2. The desktop application must register itself as an EweSync AutoStart entry so that when a EweSync connection is made, it will be executed.

 

Here is a snippet of code that accomplishes that. It is for an example Contact database. The application is to be placed in a Ewe file named "Contacts.ewe", but the install location is not known. The method:

 

String Registry.getEweCommandLine(String myEweFile, String args, String vmArgs, boolean includePathToVM)

 

is used to create a command line for the installed Ewe file of an application. It retrieves the Program Directory of the application (i.e. the location of the application Ewe file) and then prepends it to the Ewe file name. The args argument is placed after the Ewe file and the vmArgs is placed before the Ewe file in the final command line. If the includePathToVM argument is true then the path to the Ewe VM will also be placed at the start of the command line.

 

In our example below, if the Contacts.ewe file is placed in "\Windows\Start Menu\Programs\Contacts" then getEweCommandLine("Contacts.ewe","sync",null,false) would return:

 

"\Windows\Start Menu\Programs\Contacts\Contacts.ewe" sync

 

//===================================================================
public static void main(String args[])
//===================================================================
{
ewe.sys.Vm.startEwe(args);
//
String syncKey = "HKEY_LOCAL_MACHINE\\Software\\EweSoft\\Contacts\\Sync";
//-------------------------------------------------
if (args.length == 0){
//-------------------------------------------------
// If run with no arguments, register the application info
// and then run normally.
//-------------------------------------------------
try{
        String applicationEweFile = "Contacts.ewe";
        String syncCommandOption = "sync";
        //
        //Make the command line to run it in Sync mode.
        //
        String commandLine = Registry.getEweCommandLine(
               applicationEweFile, syncCommandOption, null, false);
        //
        //On the desktop register as an auto-start entry.
        //
        if (!ewe.sys.Vm.isMobile())
               RemoteConnection.registerAutoStartEweApp(
                       "Ewesoft Contacts Synchronizer",commandLine);
        //
        //On the mobile register it in the syncKey for access by the desktop.
        //
        else
               Registry.getLocalKey(0,syncKey,true,true)
                       .setValue(null,commandLine);
}catch(Exception e){
        //Could not register. Maybe report the error.
}
        //
        // Now run the application normally and exit after.
        //
        ewe.sys.Vm.exit(0);
//-------------------------------------------------
}else if (args[0].equals(syncCommandOption)){
//-------------------------------------------------
// Here run it in sync mode. 
// The same command is used for desktop and mobile version.
//-------------------------------------------------
try{
        // Make the connection. Use a unique service name
        // and get the command line from the syncKey on the remote device.
        //
        ewe.net.Socket s = RemoteConnection.getConnection().makeSyncConnection(
               "ContactsSyncService",syncKey,new ewe.sys.TimeOut(30*1000),
               RemoteConnection.MOBILE_IS_EWE_APPLICATION|
               RemoteConnection.MOBILE_GET_COMMAND_FROM_REGISTRY);
               //
               // Now use the Socket to do synchronization.
               //
}catch(Exception e){
        //Failed to make the connection.
        new ewe.ui.ReportException(e, "Could not make sync connection.",null,false);
}
        ewe.sys.Vm.exit(0);
}
//===================================================================
}
//===================================================================
 

 

With this code, the only thing you need to ensure is that the application is placed in a Ewe file of the correct name. In this case the application must be placed in a file called "Contacts.ewe", but that file may be placed anywhere on the desktop or the mobile device.

 

This is all you need to do to make an application that can synchronize with its mobile counterpart. However you will have to implement the synchronization of the data yourself. You can either use remote method calls (using RemoteCall/RemoteCallHandler objects) or, if you are synchronizing Ewe databases, you can use the ewe.datastore.RemoteSynchronizer object.
