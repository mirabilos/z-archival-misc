Ewe Streaming I/O

The Ewe I/O library is similar to the Java I/O library in a number of ways, but is also extended somewhat in other ways.

 

The java.io Compatibility Classes

The standard InputStream, OutputStream, Reader, Writer classes all exist (along with a number of standard implementations) as well as the DataInput and DataOutput interfaces (again, along with a number of standard implementations). This means that for the most part, standard Java applications can be ported directly to Ewe applications by simply substituting ewe.io for instances of java.io.

 

An additional class, called RandomStream has also been provided. A RandomStream is an object that can provide both read and write operations with exactly the same signature as those in InputStream and OutputStream as well as providing extra operations that apply specifically to I/O of a random access nature - e.g. setting and determining the current read/write position. These include:

 

void seek(long position) - to move the current read/write position in the Stream.

long tell() - to get the current read/write position in the Stream.

long length() - to get the current length of the Stream.

void setLength(long size) - to extend or truncate the length of the Stream (if possible).

boolean canWrite() - to tell if the RandomStream is opened for both reading and writing.

 

Since a RandomStream provides the same read() and write() methods of InputStream and OutputStream, it would be nice to be able to use a RandomStream in operations that require either an InputStream or OutputStream. To allow for this the RandomStream object has two methods:

 

·        InputStream toInputStream() - to provide an InputStream that uses the RandomStream for input (starting at the current read/write position).

·        OutputStream toOutputStream() - to provide an OutputStream that uses the RandomStream for output (starting at the current read/write position).

 

This brings us to an important point. The InputStream and OutputStream class as they stand in the java.io library have no real purpose other than to be extended to provide actual streaming input and output. You would never construct an InputStream or OutputStream directly under Java since they read and write to nothing.

 

However under Ewe the InputStream and OutputStream classes also provide the function of adapting other classes to act as an InputStream or OutputStream. For example their constructors include: InputStream(RandomStream rs) and OutputStream(RandomStream rs). So constructing these classes with a RandomStream as a parameter will actually produce an InputStream or OutputStream that reads from or writes to that RandomStream. So the method toInputStream() and toOutputStream() in RandomStream are actually implemented as simply:

 

//===================================================================
public InputStream toInputStream()
//===================================================================
{
  return new InputStream(this);
}
//===================================================================
public OutputStream toOutputStream()
//===================================================================
{
  return new OutputStream(this);
}

 

Converting Between java.io Classes and ewe.io.Stream Classes

 

Ewe also provides an additional Streaming interface called ewe.io.Stream and its extension ewe.io.RandomAccessStream. These streams (which are actually interfaces) pre-date the introduction of InputStream and OutputStream in the Ewe library and were used to provide bi-directional blocking and non-blocking streaming I/O. Although you would hardly use and create these streams directly, they are returned by a number of Ewe library methods and are used as parameters for a number of methods as well.

 

Luckily there are methods in Ewe that allow you to easily provide a Stream from an InputStream/OutputStream/RandomStream object as well as to convert a Stream to an InputStream or OutputStream.

 

  • Stream toReadableStream() is a method in InputStream and RandomStream that will provide a Stream object from these two classes. That Stream can only be used for reading. Attempting to write to the returned Stream will generate an error.

  • Stream toWritableStream() is a method in OutputStream and RandomStream that will provide a Stream object from these two classes. That Stream can only be used for writing. Attempting to read from the returned Stream will generate an error.

  • RandomAccessStream toRandomAccessStream() is a method in RandomStream that will provide a RandomAccessStream object.

  • You can use the StreamAdapter constructor: StreamAdapter(InputStream in, OutputStream out) to create a bi-directional Stream from an InputStream and OutputStream.

 

Also there are methods toInputStream() and toOutputStream() in the Stream interface to allow the creation of InputStream and OutputStream objects to read and write to the Stream, as well as a toRandomStream() method in RandomAccessStream to provide a RandomStream object.

 

The Stream Interface

The Stream interface shares a number of methods InputStream and OutputStream. These are standard  methods that would be familiar to Java programmers.

 

int read(); - to read a single byte.

int read(byte [] buffer); - to read an array of bytes.

int read(byte [] buffer, int startIndex, int length); - to read a section of an array of bytes.

 

All these methods throw an IOException on error and return -1 if the end of the stream has been read. The second two methods return the number of bytes actually read, while the first one returns the byte in the lower 8 bits of the return value (if -1 is not returned).

 

All these methods block the current mThread until data is available. Other mThreads can operate while these methods block.

 

void write(int value); - to write a single byte.

void write(byte [] buffer); - to write an array of bytes.

void write(byte [] buffer, int startIndex, int length); - to write a section of an array of bytes.

 

All these methods throw an IOException on error, and all will block until all the bytes have been written.

 

Additionally there are methods for close() and flush() to close the stream and to flush any buffered data to the stream.

The RandomAccessStream Interface

This interface provides the same read and write methods as Stream but additionally has methods for dealing with determining and setting the read/write position.

 

void seek(long position);

long tell();

long length();

void setLength(long size)

boolean canWrite()

 

These methods all throw an IOException if an error occurs during their operation.

Blocking vs Non-blocking I/O Methods

All of the methods mentioned so far are blocking methods. They block the current mThread until data has been read or written, but they allow other mThreads to operate. They will also operate under non-mThread threads, but if they block in this case they will block the entire application.

 

Stream and RandomAccessStream provide fully non-blocking versions of all of these methods, but you would hardly need to use them. Generally, if you do not wish your application to be blocked by I/O, you would create a new mThread to handle I/O for a particular Stream. However if you are creating a new Stream class then you would need to provide methods for these non-blocking methods. This is described in a later section.

Getting Streams to Files and Memory

 

Streams to Files

A ewe.io.File object is used to represent a File on a file system. It is explained in detail in a following chapter, but at this stage we can say that you create a File using the constructor:

 

File(String pathName); or File(File parentDirectory, String pathName);

 

You do not use a File for direct streaming I/O, but rather you can get a Stream for input or output using the following methods:

 

·        Stream toReadableStream();

·        Stream toWritableStream(boolean append);

·        RandomAccessStream toRandomAccessStream(String mode);

 

The method toRandomAccessStream() takes a parameter called mode that should be either "r" or "rw", indicating whether you want read-only or read-write access to the file. These methods all throw an IOException if a stream could not be created for the particular operation.

 

You can also create a RandomAccessFile directly giving it a pathname or File, using the constructors:

 

RandomAccessFile(String pathName, String mode);

RandomAccessFile(File file, String mode);

 

Again, mode should be "r" or "rw". If the file does not exist when opening in "rw" mode, it will be created.

 

You can also create a FileInputStream() and FileOutputStream() object directly to get an InputStream or OutputStream for a File.

 

Streams to Byte Arrays

You can use any of the classes ByteArrayInputStream, ByteArrayOutputStream and ByteArrayRandomStream to create Streams that read and write to an array of bytes in memory. These classes closely mirror those in the standard Java IO library.

In-memory Streams

The MemoryStream class is used for two purposes:

 

1.      It can be used as an in-memory one-way data pipe, allowing data to be written using a Stream interface and then read out again by another object in the application also via a Stream interface.

2.      It can be used to provide a Stream interface for reading from a non-streaming data source, or for writing to a non-streaming data sink.

Using the default constructor MemoryStream() the MemoryStream is setup as an in-memory data pipe. Any data written using the various write() methods can then be read out again using the read() methods. Note that this is different to ByteArrayInputStream and ByteArrayOutputStream. You would usually use this by having one object write into the MemoryStream while another object reads from it.

 

There is a static method called Object[] pipe() which creates a MemoryStream, splits it into an InputStream and an OutputStream and returns both the InputStream and OutputStream in an array of two Objects. The object at index 0 of the returned array is the InputStream while that at index 1 is the OutputStream. All data written to the OutputStream will be read in from the InputStream. You would then provide the object that is providing data with the OutputStream and the object that needs to read data with the InputStream.

 

There is another static method called Stream[] pipe2(). This creates two MemoryStreams and then adapts their Input and Output  streams to produce and return two equal bi-directional Stream objects. All data written to one stream will be read from the other stream and vice-versa. This provides two-way communications via two separate Stream objects. This can be used to substitute for a Socket, for example, when testing client-server applications.

 

Using the MemoryStream to provide a Streaming interface requires you to setup the MemoryStream either in read-only or write-only mode and then to override certain key methods for providing or using data. Refer to the API documentation on MemoryStream to see how to do this.

Transfering Data Between Streams

 

The ewe.io.StreamUtils Class

This class provides a number of static methods that can be used with InputStream and OutputStream classes. These fall into these categories:

 

·        readFully() methods, used to read an exact number of bytes from an InputStream. These methods will throw an EOFException if the InputStream ends before the full number of bytes have been read.

·        readAll() methods, used to read all or a fixed number of bytes of an InputStream into memory (into a ByteArray).

·        transfer() methods, used to transfer all or a fixed number of bytes from an InputStream into an OutputStream.

The ewe.io.IOTransfer Class

 

This class has been replaced by the methods provided in the StreamUtils class and you should not use it unless you have a specific reason for doing so.

 

The IOTransfer class can also be used to transfer data from one Stream to another. This can either be done synchronously (blocking) or asynchronously (non-blocking).

 

A quick way of doing this synchronously is:

 

new IOTransfer().transfer(in,out);

 

This transfers synchronously and will not return until the operation is complete or an IOException is thrown. Running this in an mThread allows other mThreads to run.

 

The do a transfer asynchronously use the constructor:

 

IOTransfer(Stream in, Stream out); or IOTransfer(Stream in, Stream out, int totalBytes);

 

The second constructor is used if you know how many bytes there are to transfer. In that case the IOTransfer will be able to report the progress of the transfer as a percentage value. To start the transfer and return a Handle do: startTask(). This returns a Handle that you can use to monitor and control the transfer.

Block Data IO

The Ewe IO library provides two classes: BlockOutputStream and BlockInputStream to allow for the sending of data in discrete blocks. This is very useful for applications where two processes are communicating using some sort of message passing protocol. The RemoteCallHandler, used to handle Remote Method Invocation (RMI) in Ewe, uses these classes. Basically an application assembles a single appropriately formatted message into an array of bytes and this is then written out using the BlockOutputStream. The data is then received by the BlockInputStream which does not provide any data to the receiver until a complete message has been fully received. The receiver then decodes the full message as necessary.

 

These classes are also useful because they can apply simple or advanced encryption to the data blocks being transmitted, thereby making it impossible to eavesdrop on the data or masquerade as legitimate communication party.

 

You create a BlockOutputStream by providing an underlying OutputStream as the actual communications channel.

 

Socket s = new Socket("MyServer",2000);
BlockOutputStream bos = new BlockOutputStream(s.getOutputStream());
bos.setEncryptor("OurSecretPassword");
//
// Now I can send off message that will be received as a single block.
//
bos.writeBlock(myData,0,256);

 

On the other end you create a BlockInputStream and then read in the data.

 

ServerSocket ss = new ServerSocket(2000);
Socket s = ss.accept();
BlockInputStream bis = new BlockInputStream(s.getInputStream());
bis.setDecryptor("OurSecretPassword");
//
// Now I can read in messages as discrete blocks.
//
ByteArray ret = bis.readBlock();

 

Advanced encryption is available for BlockOutput/InputStream using Public Key cryptography, such as that provided by the Ewe RSAKey class.
