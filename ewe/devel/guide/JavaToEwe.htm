Java to Ewe Issues

 

This chapter provides information to experienced Java users who are starting to program in Ewe. It provides pointers to classes that you would need but may not easily find in the API. Most other classes will be found in the appropriately named packages.

The Ewe Packages

java.lang

This is only a subset of the standard java.lang package. Only certain classes and certain methods in each class are implemented. Always check the ewe API to see if the class you want to use is there. If not, it may be implemented in another class in a different package.

ewe.data

This provides general purpose data interfaces and classes, along with utility methods.

ewe.datastore

This provides the implementation for the portable Ewe Database API.

ewe.filechooser

The FileChooser and ImageFileChooser classes are the standard file chooser dialog boxes. The ImageFileChooser is used to show previews of the images during selection.

ewe.fx

This provides multimedia primitives (similar to java.awt) including the base graphics for drawing. The Graphics class along Image, mImage, Brush and Pen are particularly useful.

ewe.graphics

This provides higher level graphics classes, including animated images and the Mosaic/InteractivePanel classes, which can be used as a base for simple 2D gaming.

ewe.io

This provides the Stream class and all supporting and derived classes. Stream objects provide most of the I/O in Ewe. The ewe.io.IO class also provides some useful methods that work on all Streams. RemoteCall and RemoteCallHandler provide an implementation of Remote Method Invocation (RMI) over Ewe Streams. Serial Port support is also provided in this package.

ewe.net

This provides the TCP/IP implementation, virtually identical to the java.net package. This also provides Infra-Red access on devices with such a port.

ewe.reflect

This is used to implement reflection in Ewe. The API is different from standard Java API but similar enough to be easy to use. The main difference is that, while the java.lang.Class Class can provide some reflection methods under Ewe, the ewe.reflect.Reflect class is the main class to use for Field/Method/Constructor discovery. Additionally a single ewe.reflect.Wrapper class is provided for passing Field/Method data and parameters.

ewe.sys

This provides a number of System Level classes and methods. All multi-threaded implementation classes are provided in this class as well. Many of these are explained in a later section.

ewe.ui

This is main GUI package, similar to java.awt, but far more powerful.

ewe.util

Similar to the java.util package; provides a number of useful utility and collection classes. The ewe.util.Utils class provides a number of useful data manipulation and sorting utilities. The ewe.util.mString class provides a number of useful String manipulation routines, including String splitting.

ewe.util.zip

This provides an API very similar to the java.util.zip API, but which is based on classes in the ewe.zip package. It is provided to make it easy to convert Java programs to Ewe.

ewe.zip

This provides Zip file support, though in an API that is somewhat dissimilar to java.util.zip. The ewe.util.zip package provides a more compatible API.

ewex.registry

This provides access to the System Registry on Win32/WinCE systems, and simulated access on Unix/Linux systems.

ewex.msencrypt – add-on package.

This provides encryption services on PocketPC and Win32 systems.

ewex.rapi – add-on package.

This provides access to the RAPI provided by ActiveSync on Win32 systems.

System Level Classes

Multi-threading Classes

Ewe uses a threading model that is different to the Java threading model, and so the java.lang.Thread and associated classes are not used. Instead you use ewe.sys.mThread to represent a Thread of execution, ewe.sys.mThreadGroup for grouping mThreads and the ewe.sys.Lock class for synchronizing mThreads. There are other useful multi-threading classes including ewe.sys.Handle, ewe.sys.Task and ewe.sys.TaskObject.

System/Runtime Access

Most System level methods are in the ewe.sys package, with ewe.sys.Vm providing most of the system level access. This does most of the work normally found in java.lang.System (which is not present in Ewe). Vm provides methods for:

Executing Processes

The exec() methods and ewe.sys.Process class are used for this.

Accessing Standard I/O Streams

The getStandardStream() method is used to get one of the standard input/output streams.

Accessing System Properties

The getParameter()/setParameter() and getProperty()/setProperty() methods are used for these.

Accessing Application Resources

openRandomAccessStream() and open/readResouce() are used for these.

Loading Dynamic Link Libraries

loadLibrary() is used for this.

Portable Exception Handling

Exceptions in Java 1.4 introduced the concept of exception chaining. The native Ewe VM also supports this, but Java 1.1, 1.2 and 1.3 do not. The Vm class provides static methods that allow you to chain exceptions together if supported on the running VM, and which fail softly without generating run-time errors if not supported. These methods include: setCause()/getCause() to set and get the cause of an exception. And getStackTrace() and getAStackTrace() to get the stack trace of an exception as a String.

Memory Management

These routines work only on a native Ewe VM as they are not provided for on a Java system. countObjects() counts the number referenced objects and getUsedMemory() indicates how much memory is being used for objects. getClassMemory() indicates how much class memory is being used. getMemoryStatus() can indicate the memory state of the computer.

Miscellaneous System Routines

copyArray() is a method that is used to copy array elements. getTimeStamp() returns a millisecond precision timestamp for the application.

Native Level String Manipulation

These are designed to provide faster String manipulation without creating new Strings or objects. getStringChars() will provide direct access to the character array of a String – under a Java VM this returns a copy of the String. mutateString() is used to mutate a String without creating a new one – under a Java VM a new one will be created.

Data Conversion

The ewe.sys.Convert class handles most data conversions that are handled in Java in individual wrapper objects.

Classes Representing Primitive Values

The ewe.sys.Long class can be used as a general object wrapper for any integer or boolean value. The ewe.sys.Double class can be used for double and float values.

Date/Time Classes

These are represented by ewe.sys.Time/ewe.sys.Date and TimeOfDay/DayOfYear.

Locale

Locale dependent data manipulation is done by ewe.sys.Locale.
